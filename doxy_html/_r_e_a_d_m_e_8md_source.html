5 # ArnLib - Active Registry Network.
6 
7 This Qt based library makes it easy to distribute changing data objects. It also gives a central place to find all your systems current data. By using the ArnBrowser, all data objects are real time presented in a tree view.
8 
9 ### Comparison to similar concepts
10 
11 * **Data mart:** Statistical data gathered from different systems. This makes it possible to run cross system analysis.
12 
13 * **Windows Registry:** Centralized configuration data. All in one place easily shared.
14 
15 * **ArnLib:** Hot changing data from different systems. Enables easy cross system data exchange, debugging etc.
16 
17 
18 ### Main features
19 
20 * Based on QT, multiple plattform and OS support.
21 
22 * Hierchical storage of "hot" changing data objects.
23 
24 * Data objects can be: integers, floats, strings, bytearrays and variants (most QT types, ex picture).
25 
26 * Data objects can be: measures, settings, datastreams, documents, scripts (js), ...
27 
28 * Data objects are thread-safe.
29 
30 * Data objects can be shared in a single program, among threads or between programs at different computers.
31 This division of program modules can be changed and are transparent to ArnLib.
32 
33 * Native support for data validation and double direction pipes.
34 
35 * Support for temporary session data objects.
36 Optional auto-delete of objects when tcp/ip close and unique uuid names.
37 
38 * Monitoring of newly created data objects and any mode change.
39 
40 * Dependency system with custom offered services and getting signals when all needed is available.
41 
42 * Optional persistent storage of object in SQLight or in a file.
43 
44 * Support for version control of objects stored in files.
45 
46 * Native support for JavaScript, jobstack of JS with preemtive and
47 cooperative script running.
48 
49 * Support for managing changed script restart in jobstack.
50 
51 * Service Api, for calling routines anywhere in connected Arn.
52 Remote procedure call made simple as "remote signal slots".
53 
54 * All data streams can easily be monitored and manual testdata can be inserted (see ArnBrowser).
55
