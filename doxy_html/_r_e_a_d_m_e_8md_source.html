5 # ArnLib - Active Registry Network.
6 
7 This Qt based library makes it easy to distribute changing data objects. It also gives a
8 central place to find all your systems current data. By using the ArnBrowser, all data
9 objects are real time presented in a tree view.
10 
11 ### Comparison to similar concepts
12 
13 * **Data mart:** Statistical data gathered from different systems. This makes it possible
14 to run cross system analysis.
15 
16 * **Windows Registry:** Centralized configuration data. All in one place easily shared.
17 
18 * **ArnLib:** Hot changing data from different systems. Enables easy cross system data
19 exchange, debugging, etc.

29 ## Main features
30 
31 * Based on QT, multiple platform and OS support.
32 
33 #### Arn Data Objects
34 
35 * Hierarchical storage of "hot" changing data objects.
36 
37 * _Arn Data objects_ can be: integers, floats, strings, byte arrays and variants
38 (most QT data types, e.g. QImage).
39 
40 * Data objects can typically be: measures, settings, datastreams, documents, scripts (js), ...
41 
42 * _Arn Data objects_ are thread-safe.
43 
44 * Native support for data validation and double direction pipes.
45 
46 #### Sharing
47 
48 * Data objects can be shared in a single program, among threads or between programs at
49 different computers. This division of program modules can be changed and are transparent
50 to usage of ArnLib.
51 
52 * Support for temporary session data objects.
53 Optional auto-delete of objects when tcp/ip closes and unique uuid names.
54 
55 * Dependency system with custom offered services and getting signals when all needed services
56 are available.
57 
58 * Monitoring of newly created data objects and any mode change.
59 
60 #### Persistent storage
61 
62 * Optional persistent storage of object in SQLight or in a file.
63 
64 * Support for version control (VCS) of objects stored in files.
65 
66 #### Java Script
67 
68 * Native support in JavaScript for: _Arn Data Objects_, Dependency system and
69 Monitoring of changed objects.
70 
71 * Java Script jobstack with preemptive and cooperative scripts running at different priorities.
72 
73 * Hot swap of changed Java Script in jobstack.
74 
75 #### Data streams and _Remote Procedure Call_
76 
77 * All data streams (pipes) can easily be monitored and manual test data can be inserted
78 (see ArnBrowser).
79 
80 * Service Api, for calling routines anywhere in connected Arn.
81 _Remote Procedure call_ (RPC) made simple as "remote signal slots".
82 
83 * Service Api has an automatically generated help for giving syntax when doing debug manual
84 typed calls to a RPC service.
