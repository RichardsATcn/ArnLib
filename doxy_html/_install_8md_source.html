4 Introduction {#ins_intro}
5 ------------
6 
7 This software uses qmake to build all its components.
8 qmake is part of a Qt distribution.
9 
10 qmake reads project files, that contain the options and rules how to
11 build a certain project. A project file ends with the suffix "*.pro".
12 Files that end with the suffix "*.pri" are included by the project
13 files and contain definitions, that are common for several project files.
14 
15 The first step is to edit the *.pri / *.pro files to adjust
16 them to your needs. Take care to select your deployment directories.

21 -------------

32  > cd doc/latex <Br>

41 Building ArnLib {#ins_build}
42 ---------------
43 
44 The software can be built both by command line and IDE (Qt Creator).
45 When using IDE, don't forget the "make install" step.
46 
47 
48 ### A) Unix
49 
50 > qmake <Br>
51 > make <Br>
52 > make install
53 
54 The easiest way of installing this library, is to let it be placed in a standard location for librarys and includes, e.g. /usr/lib and /usr/include/ArnLib.
55 When using a shared library it's path has to be known to
56 the run-time linker of your operating system. On Linux systems read
57 "man ldconfig" ( or google for it ). Another option is to use
58 the LD_LIBRARY_PATH (on some systems LIBPATH is used instead, on MacOSX
59 it is called DYLD_LIBRARY_PATH) environment variable.
60 
61 If you only want to check the library examples without installing something,
62 you can set the LD_LIBRARY_PATH to the lib directory
63 of your local build.
64 
65 The examples is built this way:
66 > cd examples/ArnDemoChat <Br>
67 > qmake <Br>
68 > make

137 Using ArnLib {#ins_usage}
138 ------------
139 
140 In the *.pro file of the application the follwing can be used:
141 > win32:CONFIG(release, debug|release): LIBS += L$$OUT_PWD/../ArnLib/release/ -lArn <Br>
142 > else:win32:CONFIG(debug, debug|release): LIBS += -L$$OUT_PWD/../ArnLib/debug/ -lArn <Br>
143 > else:unix: LIBS += -L$$OUT_PWD/../ArnLib/ -lArn <Br>
144 >
145 > INCLUDEPATH += $$PWD/..
146 
147 This will give a starting point for the configuration. It works well when using the same base directory for ArnLib as the application, e.g. basedir/ArnLib and basedir/myApp. In Unix alike systems it's also needed to install the library files in a path known by the system, see a) Unix.
148 
149 If you don't use qmake you have to add the include path to find the ArnLib
150 headers to your compiler flags and the ArnLib library to your linker list.
