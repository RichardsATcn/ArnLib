4 Introduction {#ins_intro}
5 ------------
6 
7 This software uses qmake to build all its components.
8 qmake is part of a Qt distribution.
9 
10 qmake reads project files, that contain the options and rules how to
11 build a certain project. A project file ends with the suffix "*.pro".
12 Files that end with the suffix "*.pri" are included by the project
13 files and contain definitions, that are common for several project files.
14 
15 The first step is to edit the *.pri / *.pro files to adjust
16 them to your needs. Take care to select your deployment directories.

42 Building ArnLib {#ins_build}
43 ---------------
44 
45 The software can be built both by command line and IDE (Qt Creator).
46 When using IDE, don't forget the "make install" step.
47 
48 
49 ### A) Unix
50 
51 > qmake <Br>
52 > make <Br>
53 > make install
54 
55 The easiest way of installing this library, is to let it be placed in a standard location
56 for librarys and includes, e.g. /usr/lib and /usr/include/ArnInc.
57 When using a shared library it's path has to be known to
58 the run-time linker of your operating system. On Linux systems read
59 "man ldconfig" (or google for it). Another option is to use
60 the LD_LIBRARY_PATH (on some systems LIBPATH is used instead, on MacOSX
61 it is called DYLD_LIBRARY_PATH) environment variable.
62 
63 If you only want to check the library examples without installing something,
64 you can set the LD_LIBRARY_PATH to the lib directory
65 of your local build.
66 it's also possible to compile the sources together by ArnLibCompile (see Using ArnLib below).
67 
68 The examples is built this way:
69 > cd examples/ArnDemoChat <Br>
70 > qmake <Br>
71 > make

143 Using ArnLib {#ins_usage}
144 ------------
145 In the *.pro file of the application the below lines can be used.
146 
147 This will give a starting point for the configuration. It works well when using the same
148 base directory for ArnLib as the application, e.g. basedir/ArnLib and basedir/myApp. In Unix
149 alike systems it's also needed to install the library files in a path known by the system,
150 see a) Unix.
151 
152 It's possible to include the ArnLib source in the application compiling by adding
153 ArnLibCompile to CONFIG. The included part of the source can be selected by addings to ARN,
154 e.g. ARN += server.
155 
156 Internal mDNS (ZeroConfig) is selected by adding mDnsIntern to CONFIG.
157 
158  CONFIG += ArnLibCompile
159  CONFIG += mDnsIntern
160 
161  greaterThan(QT_MAJOR_VERSION, 4) {
162  ARNLIB = Arn5
163  } else {
164  ARNLIB = Arn4
165  }
166 
167  ArnLibCompile {
168  #ARN += client
169  ARN += server
170  ARN += discover
171  include(../ArnLib/src/ArnLib.pri)
172  INCLUDEPATH += $$PWD/../ArnLib/src
173  } else {
174  win32: INCLUDEPATH += $$PWD/../ArnLib/src
175  win32:CONFIG(release, debug|release): LIBS += -L$$OUT_PWD/../ArnLib/release/ -l$${ARNLIB}
176  else:win32:CONFIG(debug, debug|release): LIBS += -L$$OUT_PWD/../ArnLib/debug/ -l$${ARNLIB}
177  else:unix: LIBS += -L$$OUT_PWD/../ArnLib/ -l$${ARNLIB}
178  }
179 
180  !mDnsIntern {
181  win32:CONFIG(release, debug|release): LIBS += -ldns_sd
182  else:win32:CONFIG(debug, debug|release): LIBS += -ldns_sd
183  else:unix: LIBS += -ldns_sd
184  }
185 
186 If you don't use qmake you have to add the include path to find the ArnLib
187 headers to your compiler flags and the ArnLib library to your linker list.
